The first, formal part of an entity's passport is a list of variables, the same for entities of all types.
Before a specific instance of an entity (a person, a task, a community) can be used, a value must be set for every variable. The values can be 0, +1, -1.

If it is a task, then
+1 == a very much needed ability
-1 == an ability forbidden to use

If it is a community, then
+1 == an ability that must be demonstrated
-1 == an ability forbidden to demonstrate

If it is a person, then
+1 == a great deal of this ability
-1 == none of this ability at all

If a person meets a task, or a person meets a community,
the calculation should proceed as follows:

Multiply the values of each parameter, taking one number from the person and another from the task or community.

Only the variables whose product is negative move on to the next round of analysis.

These variables are split into those where the person has a plus (a great deal of ability)
(and therefore the task or community has minus one here),
and those where the person has a minus — the ability is required but the person doesn't have it — FAILURE.

That is, there is plenty of the ability, but it is forbidden to use. Such a pair is called a CATASTROPHE. If there are several options, or if it is possible to abstain from choosing,
a catastrophe immediately blocks that option.

If it is not possible to refuse, one can only recommend that the person use maximum self-control, but this won't last long anyway.

If there remain options with no catastrophes, the ones chosen are those with the fewest failures.

If there are still tied options at this point, the ones chosen are those with the maximum number of ++ pairs, i.e. abilities that are plentiful and in demand.
